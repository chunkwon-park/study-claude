You translate a (potentially large) JSON file's natural-language content into Korean **without ever breaking the JSON structure**. The core principle: an LLM must NEVER regenerate the JSON skeleton. You only mutate leaf string values, so keys, nesting, schemas, types, and escaping stay mathematically intact.

## Arguments

`$ARGUMENTS`

The first token is the path to the input JSON file (e.g. `send.json`, `/abs/path/data.json`). If no path is given, ask the user which file. Resolve it relative to the current working directory if it is not absolute.

## Output

Write the translated file next to the source as `{basename}-ko.json` (e.g. `send.json` → `send-ko.json`, `data.json` → `data-ko.json`). Preserve pretty-printing with 2-space indent.

## Why the naive approach fails

Feeding the whole JSON blob to an LLM to "translate" makes it regenerate the structure — quotes, braces, escapes, and schema keys get mangled, especially for large files. We avoid this entirely by separating **structure** from **translatable text**.

## Workflow

### 1. Analyze and extract translatable leaf strings (by path)

Run a Node script to parse the JSON and pull out only the human-language string leaves, each with its JSON path. This is the Claude-API-payload shape (`system`, `messages`, optionally `tools`); adapt the targets if the file differs.

```bash
node -e '
const fs=require("fs");
const file=process.argv[1];
const d=JSON.parse(fs.readFileSync(file,"utf8"));
const items=[];
// system: array of {text} blocks OR a plain string
if(Array.isArray(d.system)) d.system.forEach((s,i)=>{ if(s&&typeof s.text==="string") items.push({path:["system",i,"text"],text:s.text}); });
else if(typeof d.system==="string") items.push({path:["system"],text:d.system});
// messages: content is string OR array of blocks with .text
if(Array.isArray(d.messages)) d.messages.forEach((m,mi)=>{
  if(typeof m.content==="string") items.push({path:["messages",mi,"content"],text:m.content});
  else if(Array.isArray(m.content)) m.content.forEach((c,ci)=>{ if(c&&typeof c.text==="string") items.push({path:["messages",mi,"content",ci,"text"],text:c.text}); });
});
// OPTIONAL — tools[].description. Off by default (large). Enable only if user asks for full translation.
// if(Array.isArray(d.tools)) d.tools.forEach((t,ti)=>{ if(t&&typeof t.description==="string") items.push({path:["tools",ti,"description"],text:t.description}); });
fs.writeFileSync("strings.json",JSON.stringify(items,null,2));
items.forEach((it,i)=> fs.writeFileSync("str_"+i+".txt", it.text));
items.forEach((it,i)=> console.log(i, JSON.stringify(it.path), "len:", it.text.length));
console.log("total:", items.length, "chars:", items.reduce((a,b)=>a+b.text.length,0));
' "{FILE}"
```

Default scope is **messages + system only**. The `tools[].description` block is commented out because it is usually huge (hundreds of KB). Only enable it if the user explicitly asks to translate everything.

### 2. Read each extracted string and translate

`Read` each `str_N.txt`. For each one, write the Korean translation to `str_N_ko.txt` using `Write`.

**Translation rules — preserve, do NOT translate:**
- Code blocks, inline code (`` `...` ``), file paths, URLs, email addresses
- XML/markup tags (`<system-reminder>`, `<operating_principles>`, etc.)
- Identifiers: skill names, agent names, tool names, slugs (`oh-my-claudecode:executor`, `claude-code-guide`), env var names, model IDs, command names (`/team`, `omc update`)
- `(Tools: ...)` lists and similar literal enumerations
- Strings that are **not natural language** (e.g. billing headers like `x-anthropic-billing-header: ...`) → copy verbatim
- Text that is **already Korean** → copy verbatim
- Leading/trailing whitespace and newlines of the original string (match them exactly — many strings start with `\n` or end with one or two `\n`)

Translate the **prose** around those preserved tokens into natural Korean. For strings that need no change (non-natural-language or already Korean), just `cp str_N.txt str_N_ko.txt`.

For very large strings, you may delegate individual strings to parallel sub-agents (one string each) to speed things up — but each must follow the rules above and write `str_N_ko.txt`.

### 3. Reinject by path and validate

Overwrite each leaf value by its path, write the output, then **diff against the original to prove only the intended leaves changed**.

```bash
node -e '
const fs=require("fs");
const file=process.argv[1];
const out=file.replace(/\.json$/i,"")+"-ko.json";
const d=JSON.parse(fs.readFileSync(file,"utf8"));
const items=JSON.parse(fs.readFileSync("strings.json","utf8"));
const set=(o,p,v)=>{let x=o;for(let i=0;i<p.length-1;i++)x=x[p[i]];x[p[p.length-1]]=v;};
items.forEach((it,i)=> set(d,it.path,fs.readFileSync("str_"+i+"_ko.txt","utf8")));
fs.writeFileSync(out,JSON.stringify(d,null,2));
// validate: re-parse + structural diff
const a=JSON.parse(fs.readFileSync(file,"utf8"));
const b=JSON.parse(fs.readFileSync(out,"utf8"));
const diffs=[];
(function cmp(x,y,p){
  if(typeof x!==typeof y||Array.isArray(x)!==Array.isArray(y)){diffs.push(p+" [type]");return;}
  if(x&&typeof x==="object"){const ka=Object.keys(x),kb=Object.keys(y);
    if(ka.length!==kb.length||ka.some((k,i)=>k!==kb[i]))diffs.push(p+" [keys]");
    ka.forEach(k=>cmp(x[k],y[k],p+"."+k));}
  else if(x!==y)diffs.push(p);
})(a,b,"$");
console.log("wrote", out, fs.statSync(out).size.toLocaleString(),"bytes");
console.log("changed leaf paths ("+diffs.length+"):"); diffs.forEach(x=>console.log("  "+x));
' "{FILE}"
```

**Verify**: every changed path must be one you intended to translate. A `[keys]` or `[type]` diff means structure broke — investigate before reporting success. Untranslated strings (verbatim copies) correctly show no diff.

### 4. Clean up temp files

```bash
rm -f str_*.txt strings.json
```

Report briefly:

- Number of strings translated vs kept verbatim
- Confirmation that the structural diff shows ONLY intended leaves changed (key/type structure 100% preserved)
